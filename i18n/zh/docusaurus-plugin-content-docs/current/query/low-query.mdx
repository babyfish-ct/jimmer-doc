title: 低代码查询
---

低代码查询是面向 Kotlin 项目的 KSP 辅助工具。它读取 Jimmer 实体上的查询注解，生成类型安全的查询函数，让后台列表页常见的动态 `where` 和 `orderBy` 规则不必在每个 Controller 中重复手写。

它并不替代常规 SQL DSL。如果每个接口都有独立的查询形状，仍应直接使用普通 DSL；如果很多列表页都遵循同一套模式，例如可选等值过滤、关键词搜索、范围查询、`in` 查询和默认排序，则适合使用低代码查询。

## 依赖配置

```kotlin
plugins {
    kotlin("jvm")
    id("com.google.devtools.ksp")
}

dependencies {
    implementation("org.babyfish.jimmer:jimmer-low-query-annotations:<jimmer-version>")
    ksp("org.babyfish.jimmer:jimmer-low-query-processor:<jimmer-version>")
}
```

默认生成包名为 `<entity-package>.generated.lowquery`，也可以显式指定：

```kotlin
ksp {
    arg("jimmerLowQuery.generatedPackage", "com.example.generated.lowquery")
}
```

## 为实体添加注解

```kotlin
import org.babyfish.jimmer.lowquery.annotation.Eq
import org.babyfish.jimmer.lowquery.annotation.Keyword
import org.babyfish.jimmer.lowquery.annotation.Like
import org.babyfish.jimmer.lowquery.annotation.OrderByDesc
import org.babyfish.jimmer.sql.Entity
import org.babyfish.jimmer.sql.Id

@Entity
interface Book {
    @Id
    @OrderByDesc
    val id: Long

    @Like
    @Keyword
    val name: String

    @Eq
    val storeId: Long?
}
```

对于上面的实体，处理器会生成 `KMutableRootQuery.ForEntity<Book>` 扩展函数。列表接口可以在普通 Jimmer 查询中调用它：

```kotlin
import demo.generated.lowquery.query

fun findBooks(
    name: String?,
    storeId: Long?
): List<Book> =
    sqlClient
        .createQuery(Book::class) {
            query(name = name, storeId = storeId)
        }
        .execute()
```

可空参数会被动态谓词跳过。例如 `name = null` 时，不会生成 `name` 相关条件。

## 查询操作符

低代码查询支持以下注解：

| 注解 | 生成条件 |
| --- | --- |
| `@Eq` / `@Ne` | 等值 / 不等值 |
| `@Like` | `ilike(..., LikeMode.ANYWHERE)` |
| `@StartsWith` / `@EndsWith` | 前缀 / 后缀 `like` |
| `@Gt` / `@Ge` / `@Lt` / `@Le` | 大小比较 |
| `@In` / `@NotIn` | 集合包含 / 不包含 |
| `@Between` | 下界和上界参数 |
| `@TimeRange` | 开始时间和结束时间参数 |
| `@Keyword` | 加入共享 `keyword` OR 模糊条件 |
| `@OrderByAsc` / `@OrderByDesc` | 生成 `orderBy` 表达式 |
| `@JimmerFindBy` | 生成额外的 `findBy...` 辅助函数 |

可以在实体上使用 `@JimmerLowQuery` 定制生成函数名、可见性和 fetcher 策略。

## 隐式等值字段

只要实体上至少存在一个低代码查询注解，没有显式查询注解的标量业务字段也会自动生成可空等值参数。

基础设施字段默认不参与这个隐式规则，除非显式添加注解。典型排除字段包括 `id`、`createTime`、`updateTime`、`creator`、`creatorId`、`updater`、`updaterId`、`deleted`、`deletedTime` 和 `tenantId`。

因此，实体通常只需要标出具有特殊行为的字段：

```kotlin
@Entity
interface Device {
    @Id
    val id: Long

    @Like
    @Keyword
    val name: String

    @TimeRange(name = "createTime")
    val createTime: LocalDateTime

    val status: Int?
}
```

在这个例子中，`name` 使用模糊查询，`createTime` 使用时间范围，而 `status` 仍会被自动生成可空等值参数。

## 基于已加载实体属性查询

处理器还会生成 `KSqlClient.createLowQuery(entity)` 扩展函数。它通过 `ImmutableObjects.isLoaded(entity, "property")` 检查属性是否已加载，因此调用方可以传入只包含筛选字段的 Jimmer draft：

```kotlin
val probe = Book {
    name = "GraphQL"
    storeId = 1L
}

val books = sqlClient.createLowQuery(probe).execute()
```

只有已加载属性会被转换成查询条件。

## 泛型 Controller 的运行时 Provider

当编译 classpath 中存在 Spring `@Component` 时，处理器会为每个实体额外生成一个 `JimmerLowQueryProvider` Bean。泛型 Controller 可以注入这些 Provider，并在不导入各实体包生成函数的情况下应用低代码查询规则。

Provider 会暴露：

- 实体类型；
- 实体属性到请求参数名的映射；
- 生成查询是否写入排序；
- `apply(query, entity)`，用于实体探针查询；
- `apply(query, source)`，用于不能塞进 draft 实体的参数，例如范围参数和集合参数。
